# EPUB 解析重构与专家审查备忘录

## 审查结论（世界级代码质量视角）

### 1) 架构问题（已修复）
- 原实现把 EPUB 的 XML 解析、HTML 清洗、封面识别、路径规范化、OCR 逻辑全部堆在 `DocumentTextExtractor.kt`。
- 导致：职责过重、可测性差、回归风险高。
- **修复**：拆分为独立 parser 子模块：
  - `data/document/epub/EpubTextParser.kt`
  - `data/document/epub/EpubModels.kt`
  - `data/document/epub/EpubPathResolver.kt`
- `DocumentTextExtractor` 现在只保留“统一路由 + 通用文本处理”，职责清晰。

### 2) 正确性边界（已修复）
- 封面识别兼容多来源：
  - `metadata/meta[name=cover]`
  - manifest `properties=cover-image`
  - OPF2 `guide/reference[type=cover]`
- 路径处理支持：
  - `../`, `./`
  - `#fragment`, `?query`
  - 反斜杠路径
  - 绝对路径与相对路径
- 命名空间鲁棒：
  - 通过 `tagName().substringAfter(':')` 做 localName 匹配，降低前缀差异影响。

### 3) 内容质量（已修复）
- Spine 章节提取时跳过 `properties=nav` 的目录资源，降低 TOC 噪音。
- HTML 清洗移除 script/style/nav/footnote 等非正文内容。
- 正文优先抽取语义块标签，失败再回退 `doc.text()`。

### 4) OCR 控制与性能（已修复）
- 会话 EPUB：明确传 `enableImageOcr = false`，不做 OCR。
- 知识库 EPUB：根据助手开关决定 OCR。
- OCR 前先做图片有效性门槛（宽高与面积），减少图标噪声和无效调用。
